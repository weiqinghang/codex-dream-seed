# Cycle report

Use this order:

1. Scope, a sanitized summary of the user anchor, included task-tree count, and excluded count
   with reasons.
2. When `user_anchor.status` is `provided`, include user perception reconciliation: felt result,
   expected result, supporting evidence, counterevidence, and one result of `aligned`,
   `partially_aligned`, `conflicting`, or `insufficient_evidence`. When the status is `none`,
   state only that the user explicitly selected the default review; do not invent a feeling or
   expectation.
3. Highest-value effective practices.
4. Highest-value reusable work.
5. Detours most worth shortening.
6. Cross-project patterns and project-specific patterns.
7. New candidates, new evidence for existing candidates, conflicts, and counterexamples.
8. A small set of high-value candidates for user review.
9. Evidence gaps and items to continue observing.

Prefer a few evidence-backed conclusions over comprehensive-looking filler. Do not repeat old
conclusions without new evidence. On the first run establish a baseline and do not claim a trend.
Keep observations separate from interpretations and cite only sanitized `TASK-*` references.
Do not quote the user's raw feedback in a shareable report. Summarize only what is necessary to
understand the comparison, and do not treat technical completion as proof of user acceptance.

# Cycle report

Use this order:

1. Scope, included task-tree count, and excluded count with reasons.
2. Highest-value effective practices.
3. Highest-value reusable work.
4. Detours most worth shortening.
5. Cross-project patterns and project-specific patterns.
6. New candidates, new evidence for existing candidates, conflicts, and counterexamples.
7. A small set of high-value candidates for user review.
8. Evidence gaps and items to continue observing.

Prefer a few evidence-backed conclusions over comprehensive-looking filler. Do not repeat old
conclusions without new evidence. On the first run establish a baseline and do not claim a trend.
Keep observations separate from interpretations and cite only sanitized `TASK-*` references.
